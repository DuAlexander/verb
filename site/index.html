verb is a library for working with NURBS surfaces and curves in the browser, a server, or desktop app.

The library can be compiled for many languages, including JavaScript, C#, Python, and C++.

Geometry

Types

See all geometry types

verb supplies a large set of routines for creating common pieces of geometry. For surfaces, these include conics, lofts, surfaces of revolution, sweeps, and extrusions. For curves, these include lines, cicles, interpolating curves, arcs, polylines, and ellipses.

Loft
new verb.Extrusion( profile, axis, distance);
RevolveSurface
new verb.RevolveSurface( profile );
BezierCurve
new verb.BezierCurve( points );

Evaluation

verb supplies methods for evaluation of points and arbitrary derivatives on NURBS curves and surfaces.

var curve = new verb.BezierCurve( [p1, p2, p3, p4] );

var numDerivs = 1
	, u = 0.5;

// synchronously evaluate the point at 0.5
var point = curve.point( u );

// synchronously evaluate first derivative of the curve at 0.5
var derivs = curve.derivatives( u, numDerivs );

Tessellation

A NURBS curve or surface is an analytical function. In order to visualize it, it must be "tessellated" into a collection of lines or triangles. verb supplies naive and adaptive tessellation routines for curves and surfaces at arbitrary tolerance.

var surface = new verb.NurbsSurface(  );

surface.tessellate( tessellationOptions, function(tessellation){
	...
});

Intersection and Analysis

verb supplies many intersection of curves, surfaces, and meshes. These algorithms can be tuned any tolerance and designed to detect multiple intersections.

Intersecting a curve and surface 

var srf = new verb.PlanarSurface( base, uaxis, vaxis, ulength, vlength );
var crv = new verb.BezierCurve( [ [5.2,5.2,5], [5.4,4.8,0], [5.2,5.2,-5] ] );
var options = { tolerance: 1e-5 };

var res = verb.intersect.curveSurface( crv, srf, options );

Asynchronous or synchronous

verb's can be called either asynchronously or synchronously. This allows verb to be extended for multi-threading and remote evaluation. 

var curve = new verb.BezierCurve( [p1, p2, p3, p4] );

// tessellate the curve synchronously with default options
var pointsSync = curve.tessellate();

// tessellate the curve asynchronously (again, default options) in a background thread
curve.tessellate({}, function(points){
	...
});

Multi-threaded execution

verb can use WebWorkers if they are available in the browser. verb can automatically put them to use - potentially running multiple instances of a stateless NURBS engine. This allows expensive operations like tessellation or intersection to be elegantly handled in the background via verb's async API.

Here's an example of building an Engine and calling it directly: 

// instantiate a nurbs evaluation engine
var engine = new verb.core.Engine();

// evaluate derivatives at a point along the curve
engine.eval('rational_curve_derivs', [degree, control_points, weights, knots, u, numDerivs], function(derivs){
	... do something with the derivs ...
});

Under the hood, all of verb's geometry types automatically use a background thread when calling the asynchronous API.
